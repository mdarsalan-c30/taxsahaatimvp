# Admin wireframes

Annotated low-fidelity wireframes for the eight core admin screens (Stage 1 deliverable).
These are the contract for the Phase 1A build. Each file is one screen.

| # | Screen | File | Phase | Primary user |
|---|--------|------|-------|--------------|
| 1 | Home (command center) | [`01_home.md`](01_home.md) | 1A | Ayush, CEO |
| 2 | Pricing & offers | [`02_pricing.md`](02_pricing.md) | 1A | CEO |
| 3 | Coupons | [`03_coupons.md`](03_coupons.md) | 1A | Ayush |
| 4 | Payments | [`04_payments.md`](04_payments.md) | 1A | CEO |
| 5 | CRM pipeline | [`05_crm_pipeline.md`](05_crm_pipeline.md) | 1A | Ayush |
| 6 | CRM contact detail | [`06_crm_contact.md`](06_crm_contact.md) | 1A | Ayush |
| 7 | Analytics | [`07_analytics.md`](07_analytics.md) | 1A | Ayush |
| 8 | CA verification queue | [`08_ca_verification.md`](08_ca_verification.md) | 1B | Ayush |

Shared chrome (all screens): left sidebar (max 12 items, 2 levels), top bar with date range
toggle, lane filter (B2C / B2B / both), user menu, and notification bell wired to the action inbox.
Conventions are defined in [`../ADMIN_DESIGN_GUIDELINES.md`](../ADMIN_DESIGN_GUIDELINES.md).
